## How to Contribute

### 1. Add New Slang Words
- Edit `data/slang_dictionary.json`
- Add new entries with proper translations
- Include cultural context when possible

### 2. Report Bugs
- Use the GitHub Issues tab
- Describe the problem clearly
- Include steps to reproduce

### 3. Suggest Features
- Open an issue with your idea
- Explain why it would be helpful
- Discuss implementation if possible

### 4. Code Contributions

#### Setup
1. Fork the repository
2. Clone your fork
3. Install dependencies: `npm install`
4. Create a new branch: `git checkout -b feature/your-feature`

#### Making Changes
1. Write clean, readable code
2. Follow existing code style
3. Add tests if needed
4. Test your changes locally

#### Submitting
1. Commit your changes: `git commit -m "Add feature"`
2. Push to your fork: `git push origin feature/your-feature`
3. Open a Pull Request

## Code Style

- Use TypeScript
- Follow existing naming conventions
- Add comments for complex logic
- Keep functions small and focused

## Questions?

Feel free to open an issue for any questions!
